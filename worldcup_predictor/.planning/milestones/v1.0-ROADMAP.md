# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-06-14
**Phases:** 1–6
**Total Plans:** 10

## Overview

A live, self-updating tournament predictor in your terminal — when a match ends, within seconds the script detects it, updates Elo, re-simulates, and shows how every team's odds changed.

## Phases (v1.0 Scope)

### Phase 1: State & Elo Foundation
- **Goal:** Tournament bracket and Elo ratings can be loaded, computed, validated, and persisted across restarts via JSON files
- **Depends on:** Nothing (first phase)
- **Plans:** 2 plans ✅ Both complete

**Details:**
- K_FACTOR=60, DEFAULT_ELO=1500 constants
- `state.py`: load_teams, load_bracket, load_played, save_teams, save_played with atomic write
- `elo.py`: expected_score, update_ratings (pure function, no mutation)
- Bracket validation: unique match_ids, valid source_matches, no circular dependencies
- 32 team seed data in teams.json, 23 bracket matches in bracket.json

### Phase 2: Monte Carlo Simulation
- **Goal:** Simulate remaining knockout tournament 50,000+ times with accurate championship probabilities
- **Depends on:** Phase 1
- **Plans:** 2 plans ✅ Both complete

**Details:**
- Pure-functional Monte Carlo engine with no I/O side effects
- `run_simulation(teams, bracket, played, iterations=50000, seed=None)` — main entry point
- Round-specific sim: `_simulate_r16`, `_simulate_knockout_round` (shared for QF/SF/FINAL)
- **50K iterations in 1.268s** (PASS, 3.7x headroom over 5s gate)
- Reproducible seeds via `random.seed(seed)` inside the function

### Phase 3: Live API Integration
- **Goal:** Fetch live match results from Football-Data.org API with robust error handling
- **Depends on:** Phase 1
- **Plans:** 2 plans ✅ Both complete

**Details:**
- `fetch_raw_matches(api_key, api_url, timeout)` — HTTP GET with 3-retry loop (1s/2s/4s backoff)
- Handles HTTP 429 with Retry-After header
- `process_matches(raw, teams, bracket, aliases, played_ids)` — resolves API team names via alias lookup
- `validate_api_key()` — checks env var, exits 1 on missing/403
- **Historical note:** v1.0 used Football-Data.org; later migrated to BSD Sports Data in v2.0

### Phase 4: Main Loop & Shutdown
- **Goal:** Autonomous operation — continuous polling, match detection, re-simulation, graceful shutdown
- **Depends on:** Phase 3
- **Plans:** 1 plan ✅ Complete

**Details:**
- `while _running:` loop with signal handler (SIGINT, SIGTERM, SIGBREAK on Windows)
- First poll fires immediately (no initial 60s wait)
- `_next_poll_sleep(interval)` — deadline-based sleep in 0.5s increments
- Ctrl+C: `_running = False`, finish iteration, save state, print final probabilities, exit 0
- Hourly auto-resim: 3600s idle trigger
- Rate limiter prevents API abuse

### Phase 5: Console Output & Formatting
- **Goal:** Beautiful, colored, delta-tracking championship probabilities in terminal
- **Depends on:** Phase 2
- **Plans:** 2 plans ✅ Both complete

**Details:**
- Raw ANSI escape codes (no colorama — pure stdlib)
- `_ansi()` factory pattern: `_bold_cyan`, `_green`, `_red`, etc.
- `_supports_color()` checks `sys.stdout.isatty()`
- Delta column: ▲ green, ▼ red, = flat, — for new
- Biggest Risers and Biggest Fallers (top-3 each)
- `print_error()` to stderr
- Windows: `os.system("")` for Console Host ANSI
- 23 tests across 11 test classes

### Phase 6: CLI Interface & Polish
- **Goal:** User controls the tool via command-line flags with full usage docs
- **Depends on:** Phase 5
- **Plans:** 2 plans ✅ Both complete

**Details:**
- `_parse_args(argv=None)` — --once, --no-color, --seed N, --help
- `output.NO_COLOR = False` module-level flag
- `--once`: straight-line fetch→simulate→print→exit(0)
- `--seed N`: threaded to every `run_simulation()` call
- `--help`: 4 flags with detailed epilog
- 13 CLI tests (8 argparse + 3 NO_COLOR + 2 integration)

## Milestone Summary

**Key Decisions (v1.0):**
- Python 3.10+ with minimal dependencies (requests only)
- Football-Data.org free API (10 req/min) — **later replaced by BSD Sports Data**
- Elo rating with K=60
- Monte Carlo simulation at 50K iterations
- JSON file persistence with atomic writes
- Console-only output (no web UI for MVP)
- Pure stdlib ANSI (no colorama)
- `--once` mode skips signal handlers and state save
- `--seed` applies to every simulation call, no global `random.seed()`

**Issues Resolved:**
- Bracket DAG validation catches circular dependencies and duplicate match_ids
- Team-alias matching resolves ~91% of API team-name mismatches
- Performance: 50K sim in 1.268s (3.7x headroom)
- Windows ANSI support via `os.system("")` + `sys.stdout.reconfigure(encoding="utf-8")`

**Issues Deferred (from v1.0):**
- End-to-end verification with real Football-Data.org API key (BSD API replaced the need)
- Group stage simulation → **Completed in v2.0**
- Web dashboard → Still deferred

**Technical Debt Incurred (v1.0):**
- `test_main_loop_runs_iterations` checks for "Fetched" which was renamed to "Polling" in Phase 5
- Team-name alias matching could break on unexpected API response formats
