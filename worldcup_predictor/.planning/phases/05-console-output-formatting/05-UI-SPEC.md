# Phase 5 — Console Output & Formatting: Design Contract

> Visual and interaction contract for the terminal-UI phase. This is NOT a web frontend —
> the "UI" is ANSI-colored console output printed to stdout. Generated by gsd-ui-researcher,
> verified by gsd-ui-checker.

---

## 1. Output System

| Property | Value | Source |
|----------|-------|--------|
| Tool | Raw ANSI escape codes | CONTEXT.md D-05 |
| Component library | `src/output.py` (single module) | CONTEXT.md D-01 |
| Icon library | Unicode symbols: `▲` `▼` `⚠` `─` `█` `⚽` | UI_UX_Design.md §4 |
| Output stream | `print()` to stdout (no logging module for MVP) | CONVENTIONS.md §Logging |
| Timestamp format | `[%Y-%m-%d %H:%M:%S]` ISO 8601 | CONVENTIONS.md §Logging |
| ANSI dependency | None — Python 3.10+ on Windows 10+ supports natively | CONTEXT.md D-05 (user confirmed) |

### Output Blocks Inventory (`src/output.py`)

All functions are pure, take data, print to stdout, return `None`. No classes.

| Function | Emits When | Contract (inputs → visible block) |
|----------|-----------|------------------------------------|
| `print_header()` | Startup (once) | Banner header + seed data summary (Section 3.1) |
| `print_probability_table(probs, prev_probs)` | After each simulation | Top-5 table with QF/SF/FINAL/CHAMPION columns + inline deltas + remaining teams one-liner (Section 3.4) |
| `print_match_alert(match)` | New match detected | Highlighted match-result block with score and winner (Section 3.3) |
| `print_elo_changes(updates)` | After match → Elo update | Per-team old → new rating listing with +/- delta (Section 3.3) |
| `print_heartbeat()` | Poll cycle with no new matches | Single timestamped "no new matches" line (Section 3.2) |
| `print_delta_summary(risers, fallers)` | After each simulation | Biggest Risers / Biggest Fallers top-3 block (Section 3.5) |
| `print_shutdown_banner(probs)` | Ctrl+C → shutdown | "Final Championship Probabilities" banner + full table (Section 3.6) |
| `print_auto_refresh()` | Hourly re-sim with no new matches | "Auto-refresh simulation" one-liner |
| `print_simulation_duration(seconds)` | After sim completes | "done in 0.8s" appended to re-sim line (agent discretion — enabled by default) |

**Color/style helpers (private to `output.py`):**

| Helper | ANSI Code | Usage |
|--------|-----------|-------|
| `_dim(text)` | `\033[2m`...`\033[0m` | Timestamps |
| `_bold_cyan(text)` | `\033[1;36m`...`\033[0m` | Headers, separators |
| `_bold_yellow(text)` | `\033[1;33m`...`\033[0m` | Match alerts |
| `_green(text)` | `\033[32m`...`\033[0m` | Probability increase, Elo increase |
| `_red(text)` | `\033[31m`...`\033[0m` | Probability decrease, Elo decrease |
| `_bold_red(text)` | `\033[1;31m`...`\033[0m` | Errors |
| `_bold_green(text)` | `\033[1;32m`...`\033[0m` | Success messages, simulation done |
| `_bold_white(text)` | `\033[1;37m`...`\033[0m` | Team names in match results |

**Fallback rule:** Check `sys.stdout.isatty()` before applying any ANSI codes.
If piped/redirected: emit plain text with symbols only (`▲`, `▼`, `⚠`).
Explicit `--no-color` override is Phase 6 scope.

## 2. Console Layout Units (Character-Based Spacing)

All spacing is in character widths (monospace grid). No pixel values exist.

### Column Widths (Probability Table)

| Column | Width | Alignment | Format | Example |
|--------|-------|-----------|--------|---------|
| Rank | 3 | Right | `N.` | ` 1.` |
| Team name | 18 | Left | `{name:<18}` | `Argentina         ` |
| QF | 6 | Right | `{p:.3f}` | ` 0.838` |
| SF | 6 | Right | `{p:.3f}` | ` 0.531` |
| FINAL | 8 | Right | `{p:.3f}` | ` 0.316` |
| CHAMPION | 8 | Right | `{p:.3f}` | ` 0.178` |
| Delta | 8 | Right | `{symbol} {diff:+.3f}` | ` ▲ +0.003` |
| **Total row width** | **~61 chars** | | | |

### Section Spacing

| Element | Spacing Rule |
|---------|-------------|
| Between top-level blocks | One blank line (`print()`) |
| Between table header and rows | Dash separator line: `"-" * 51` |
| Between probability table and delta summary | One blank line |
| Within a block | Compact (no extra blank lines) |
| Separator banner width | `"=" * 60` |

### Table Frame

```
Rank Team_name          QF     SF   FINAL  CHAMPION  Delta
───────────────────────────────────────────────────────────
```

Dash separator: `"-" * 51` (legacy from existing table).
Table separator matches actual column count.

## 3. Console Typography

No font choices — the user's terminal controls font. All values are monospace fixed-width.

| Role | Property | Value |
|------|----------|-------|
| Font family | User terminal default | Monospace (Courier New / Consolas / etc.) |
| All text | Character width | 1 char = 1 monospace cell |
| Body | No size choice | User's terminal font size |
| Numeric formatting | Decimal places | Exactly 3 (`:.3f`) |
| Percentage display | Fractional | Decimal 0–1 range (e.g., `0.178` not `17.8%`) |
| Delta display | Signed | `+0.003` or `-0.002` with `:.3f` format |

**Why `:.3f` not percentage:** The PRD examples show decimal fractions matching the simulation module's output format. Keeps the display tied to the data structure. Change to `:.1%` if user preference shifts — but not for Phase 5.

## 4. Color Contract (ANSI)

### Color Palette

| Role | ANSI Code | Hex Equivalent | Element(s) Reserved For |
|------|-----------|----------------|--------------------------|
| Timestamp | `\033[2m` (dim) | Gray (#888) | All `[YYYY-MM-DD HH:MM:SS]` prefixes |
| Header/Separator | `\033[1;36m` (bold cyan) | #00FFFF | `===` banners, table header row |
| Match alert | `\033[1;33m` (bold yellow) | #FFFF00 | `NEW MATCH DETECTED!` — the ONLY element in yellow |
| Team names in match result | `\033[1;37m` (bold white) | #FFFFFF | `Argentina 2 - 1 Nigeria` team names |
| Probability / Elo increase | `\033[32m` (green) | #00FF00 | `▲` symbol, `+N.NNN` delta values, Elo `+NN` |
| Probability / Elo decrease | `\033[31m` (red) | #FF0000 | `▼` symbol, `-N.NNN` delta values, Elo `-NN` |
| Error prefix | `\033[1;31m` (bold red) | #FF0000 | `⚠` error prefix + error message text |
| Success / completion | `\033[1;32m` (bold green) | #00FF00 | `done in 0.8s`, `State saved. Goodbye.` |

### Color Rules

1. **Never apply two colors to the same string.** Each token gets one color wrapper.
2. **Color is additive to symbols, not a replacement.** `▲`/`▼`/`⚠` always present regardless of color support.
3. **Timestamp color is ALWAYS dim gray** — never change. This is the "background noise" color that makes non-timestamp content pop.
4. **Match alerts are the ONLY bold yellow element** — this is the highest-visibility event. Nothing else competes.
5. **Errors use bold red + `⚠` prefix** — never just bold red without the symbol.
6. **Delta values inherit the color of their direction symbol** — `▲` and the delta number are both green, `▼` and the delta number are both red.

## 5. Interaction Contract

This phase has ZERO user input handling. All interactions are system-initiated console writes.

| User Action | What Happens | Printed By |
|-------------|-------------|------------|
| `python main.py` | Header + initial sim + probabilities | `print_header()` + `print_probability_table()` |
| Ctrl+C (SIGINT) | Running flag → finishes iteration → shutdown banner | `print_shutdown_banner()` |
| Pipe stdout (`python main.py > log.txt`) | Plain text fallback (no ANSI) | Auto via `isatty()` check |
| SIGTERM / SIGBREAK (Windows) | Same as SIGINT | `print_shutdown_banner()` |
| Terminal resize | No effect — append-only output, no cursor management | — |

**Explicitly NOT implemented in Phase 5 (deferred to Phase 6):**

| Action | Flag | Phase |
|--------|------|-------|
| Disable colors | `--no-color` | Phase 6 |
| Single run | `--once` | Phase 6 |
| Reproducible seed | `--seed N` | Phase 6 |
| Help | `--help` | Phase 6 |

## 6. Copywriting Contract (Exact Output Format)

### 6.1 Startup Banner (`print_header()`)

```
════════════════════════════════════════════════════════════════════════
  WORLD CUP DYNAMIC PREDICTOR — MVP
  Polling API every 60 seconds. Press Ctrl+C to stop.
  Loaded {N} teams, {N} bracket matches, {N} played matches, {N} aliases.
════════════════════════════════════════════════════════════════════════
```

- `"=" * 60` for separator lines
- Bold cyan for entire block
- Team/bracket/played/alias counts come from `len()` of loaded structures

### 6.2 Initial Probabilities (first call to `print_probability_table()`)

```
[2026-06-13 22:00:00] Initial probabilities:
 1. Argentina        0.838 0.531 0.316 0.178
 2. Brazil           0.801 0.491 0.257 0.135
 3. France           0.754 0.432 0.211 0.102
 4. England          0.728 0.371 0.177 0.083
 5. Spain            0.695 0.312 0.144 0.063
 ─── 27 other teams — best: Nigeria (0.002)
```

- **First call** (no `prev_probs`): No delta column, no risers/fallers block
- Timestamp dim gray, header row bold cyan, team names plain
- "Initial" label on first call, "UPDATED PROBABILITIES:" on subsequent

### 6.3 Heartbeat (`print_heartbeat()`)

```
[2026-06-13 22:01:00] Polling... no new matches.
```

- Dim gray timestamp, rest plain
- One line per poll cycle
- **Do NOT print if a match was detected this cycle** (match alert replaces it)

### 6.4 Match Alert + Elo Changes (`print_match_alert()` + `print_elo_changes()`)

```
════════════════════════════════════════════════════════════════════════
  NEW MATCH DETECTED!
  Argentina 2 - 1 Nigeria
  Winner: Argentina
════════════════════════════════════════════════════════════════════════
[2026-06-13 22:03:00] Updating Elo:
   Argentina: 2100 → 2112  (+12)
   Nigeria:   1850 → 1838  (-12)
```

- Bold yellow for separator lines + "NEW MATCH DETECTED!" line
- Team names in match result: bold white
- Score separator: " - " (space dash space)
- Elo update line: team name left-aligned (width 12), "old → new" with (+N) or (-N) at end
- `+N` green, `-N` red
- Timestamp on the "Updating Elo:" line only

### 6.5 Updated Probabilities with Deltas (`print_probability_table()` with `prev_probs`)

```
[2026-06-13 22:03:07] Re-simulating (50000 runs)... done in 0.8s
[2026-06-13 22:03:07] UPDATED PROBABILITIES:
 1. Argentina        0.838 0.531 0.316 0.178  ▲ +0.003
 2. Brazil           0.801 0.491 0.257 0.135  ▼ -0.002
 3. France           0.754 0.432 0.211 0.102  ▲ +0.001
 4. England          0.728 0.371 0.177 0.083  ▼ -0.005
 5. Spain            0.695 0.312 0.144 0.063  ▲ +0.002
 ─── 27 other teams — best: Nigeria (0.002)
```

- "Re-simulating" line with duration: `bold_green("done in {s:.1f}s")`
- Delta symbol + value right-aligned in 8-char field
- Delta color: `▲` = green, `▼` = red (value inherits same color)
- "UPDATED PROBABILITIES:" label bold cyan
- Table header: bold cyan (same as separator line style)

### 6.6 Remaining Teams One-Liner

```
 ─── 27 other teams — best: Nigeria (0.002)
```

- `"───"` prefix (Unicode em dash ×3 + space)
- Always show count of remaining teams
- "best: {name} ({value})" — the non-top-5 team with highest champion probability
- Plain text (no color) — this is supporting information

### 6.7 Biggest Risers / Biggest Fallers (`print_delta_summary()`)

```
Biggest Risers
  ▲ Argentina   +3.2%
  ▲ Brazil      +1.8%
  ▲ Spain       +1.1%

Biggest Fallers
  ▼ France      -2.9%
  ▼ England     -1.7%
  ▼ Germany     -1.0%
```

- Label: bold cyan
- Exactly 3 risers and 3 fallers (or fewer if fewer changed)
- Delta values as **percentage** (`:.1%`) — NOT decimal fractions — because this is a summary block meant for quick scanning
- Risers green, fallers red
- Team name left-aligned (width 16), delta right-aligned
- One blank line between risers and fallers sections
- **This block only prints if `prev_probs` was provided** (i.e., not on initial run)

### 6.8 Shutdown Banner (`print_shutdown_banner()`)

```
════════════════════════════════════════════════════════════════════════
  FINAL CHAMPIONSHIP PROBABILITIES
════════════════════════════════════════════════════════════════════════
 1. Argentina        0.838 0.531 0.316 0.178
 2. Brazil           0.801 0.491 0.257 0.135
 ...
 32. Nigeria         0.002 0.000 0.000 0.000

State saved. Goodbye.
```

- Bold green banner + "State saved. Goodbye." in bold green
- Full team list (all 32, not top-5) — this is the final record
- No delta column (no `prev_probs` context at shutdown — the sim is fresh)
- No remaining-teams one-liner (all teams shown)

### 6.9 Auto-Refresh One-Liner (`print_auto_refresh()`)

```
[2026-06-13 23:00:00] Auto-refresh simulation (no new matches in 1h)
```

- Plain text with dim gray timestamp
- Followed by re-sim and probability table

### 6.10 Error Messages

Error messages should appear as natural `print()` calls, not through output.py (they originate from main.py or fetcher.py contexts):

```
[2026-06-13 22:10:00] ⚠ API error: timeout. Retry in 60s. Using cached data.
```

- Bold red for `⚠` symbol and the message
- Never more than 3 consecutive error lines (to prevent spam)
- Error format: `"⚠ {problem}. {action}."`

### 6.11 Destructive Actions

| Action | Confirmation Mechanism |
|--------|----------------------|
| Shutdown (Ctrl+C) | Prints "Shutdown requested — finishing current iteration..." then saves state. No user confirmation needed. |
| State save failure | `print()` warning to stderr, continues loop. No destructive outcome. |

**No destructive confirmations in Phase 5.** The app has no user-input prompts.

## 7. Delta Tracking Contract (`prev_probs`)

| Property | Value | Source |
|----------|-------|--------|
| Storage | In-memory variable in `main.py` iteration scope | CONTEXT.md D-02 |
| Persistence | None — lost on restart | User confirmed |
| Data structure | Same as simulation output: `dict[str, dict[str, float]]` with qf/sf/final/champion | |
| Scope | Only the `champion` key is used for delta display | Table shows only champion delta for scannability |
| Initial call | `prev_probs = None` → skip deltas, skip risers/fallers | |
| Calculation | `delta = probs[name]["champion"] - prev_probs[name]["champion"]` | |
| Threshold | No threshold — show delta for all top-5 even if 0.000 (`▸ +0.000`) | |
| Delta display | Exactly 3 decimal places | `:.3f` |

## 8. Autoscroll Behavior

| Property | Value | Rationale |
|----------|-------|-----------|
| Screen strategy | **Append only** — never clear screen, never cursor-up | User confirmed in discussion: odds evolution timeline is valuable |
| Scrollback | Full terminal history preserved | Simpler than curses; user can scroll back to see history |
| Timestamps | Every block prefix has a timestamp line | Creates a readable timeline in scrollback |
| New content position | Appended at bottom of terminal | Natural terminal behavior |
