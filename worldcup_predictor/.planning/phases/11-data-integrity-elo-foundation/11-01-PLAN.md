    ---
    **`fetch_eloratings_tsv(url: str = "") -> str | None`**
    - Default url: constants.ELORATINGS_TSV_URL
    - GET request with timeout=constants.ELO_SYNC_TIMEOUT
    - Retry 3 times with backoff from constants.ELO_SYNC_RETRY_BACKOFFS (identical pattern to fetcher.py lines 31-83)
    - Exceptions caught individually: Timeout, ConnectionError, HTTPError, RequestException
    - Returns `resp.text` (raw TSV) on success, `None` after 3 failed attempts
    - `resp.raise_for_status()` between retries

    ---
    **`parse_eloratings_tsv(tsv_raw: str) -> list[tuple[str, float]]`**
    - Pure function, no network I/O
    - Uses `csv.reader(io.StringIO(tsv_raw), delimiter="\t")`
    - Extracts col index 2 (team_code) and col index 3 (elo_rating) from each row
    - Skips empty rows
    - Converts rating to float; skips rows where rating is not a valid float
    - Returns list of `(team_code, elo_rating)` tuples

    ---
    **`validate_eloratings_data(parsed: list[tuple[str, float]]) -> tuple[bool, list[str]]`**
    - Per D-08: schema validation
    - Returns (valid: bool, messages: list[str])
    - Checks:
      1. At least 48 entries parsed (messages: f"Expected >= 48 teams, got {len(parsed)}")
      2. All ratings in [1000, 2500] range (messages: f"Rating {rating} for {code} out of range [1000, 2500]")
      3. No NaN or None values
    - Does NOT raise — returns validity flag and list of messages
    - Accept if len(parsed) could be >48 (eloratings has 244 rows). Only flag if <48.

    ---
    **`resolve_team_names(parsed: list[tuple[str, float]], teams: dict[str, dict]) -> dict[str, float]`**
    - Maps parsed (code, rating) pairs to canonical team names
    - Uses `constants.ELORATINGS_TEAM_CODES` to resolve code → canonical name
    - Only includes teams that exist in the `teams` dict (canonical_name in teams)
    - Uses case-sensitive dict key check (canonical names are capitalized proper in teams.json)
    - Logs WARNING for unmapped codes via `logger.warning("Unmapped eloratings code: %s", code)`
    - Logs WARNING for teams found in code map but not in teams dict (shouldn't happen if 48:1 coverage is maintained)
    - Returns dict of `{canonical_name: elo_rating}`

    This is where the 2 alias cases resolve automatically: ELORATINGS_TEAM_CODES["TR"]="Türkiye" matches teams.json key "Türkiye" directly (no alias lookup needed). Same for "CZ"→"Czech Republic".

    ---
    **`apply_graduated_correction(teams: dict[str, dict], eloratings_values: dict[str, float]) -> list[dict]`**
    - Per D-10 through D-13: hybrid graduated correction
    - For each canonical_name in eloratings_values:
      - `current_elo = teams[canonical_name]["elo"]`
      - `drift = eloratings_value - current_elo`
      - `abs_drift = abs(drift)`
      - If `abs_drift < ELO_DRIFT_TOLERANCE` (10): skip (D-11: ignore)
      - If `abs_drift <= ELO_BLEND_THRESHOLD` (30): `new_elo = round(current_elo + drift * ELO_BLEND_FACTOR, 1)`, reason = "blended_50pct" (D-11: blend 50%)
      - If `abs_drift > ELO_BLEND_THRESHOLD` (30): `new_elo = round(eloratings_value, 1)`, reason = "overwrite_drift_gt_30" (D-11: overwrite+flag)
      - Mutates `teams[canonical_name]["elo"] = new_elo` in-place
      - Appends correction entry dict with keys: timestamp (datetime.utcnow().isoformat()), team, old_value, new_value, source ("eloratings.net"), reason, drift_magnitude (round(drift, 1))
      - Per D-12: every correction logged
    - Uses `from datetime import datetime, timezone` for UTC timestamps (use `datetime.now(timezone.utc).isoformat()`)
    - Returns list of correction log entries

    ---
     **`sync_elo_from_eloratings(teams: dict[str, dict]) -> list[dict] | None`**
    - Full orchestrator: fetch → parse → validate → resolve → correct → persist
    - Steps:
      1. `tsv_raw = fetch_eloratings_tsv()`
      2. If None (fetch failed): return None (caller handles cache fallback per D-15/D-19/D-20)
      3. `parsed = parse_eloratings_tsv(tsv_raw)`
      4. `valid, messages = validate_eloratings_data(parsed)`
         - If not valid: log each message as warning, still continue (partial data is better than none per D-21)
      5. `eloratings_values = resolve_team_names(parsed, teams)`
      6. `corrections = apply_graduated_correction(teams, eloratings_values)`
      7. Persist audit trail: `log = state.load_elo_update_log()` → `log.extend(corrections)` → `state.save_elo_update_log(log)`
      8. Persist cache: `state.save_eloratings_cache({"fetched_at": datetime.now(timezone.utc).isoformat(), "values": eloratings_values})`
       9. If corrections exist: `state.save_teams(teams)` (persist updated Elo values)
       10. Return corrections list if drift found, or empty list `[]` if no drift, or `None` if fetch failed (caller disambiguates via cache check)
    - Log a summary: `logger.info("Sync complete: %d corrections applied", len(corrections))`
    - Log count of unresolved codes: `unresolved = len(parsed) - len(eloratings_values)`, if > 0: `logger.warning("Sync: %d team codes unresolved", unresolved)`
