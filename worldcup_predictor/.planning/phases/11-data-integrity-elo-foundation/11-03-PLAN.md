    **Class `TestParse`** (tests parse_eloratings_tsv):
    - `test_basic_tsv_parsing` — Parse 3-row TSV with valid data → returns 3 tuples, correct types
    - `test_empty_input` — Empty string → returns []
    - `test_skips_empty_rows` — TSV with blank lines → skips them
    - `test_valid_rating_types` — Rating column is parsed as float
    - `test_real_fixture` — Parses `tests/fixtures/eloratings_world.tsv` → returns 48+ tuples (live fixture may have >48 because eloratings includes non-WC teams). Use parametrize or simple `assert len(parsed) >= 48`.

    **Class `TestCorrection`** (tests apply_graduated_correction):
    - Uses conftest `sample_teams` fixture or minimal inline teams dict
    - `test_ignore_small_drift` — drift < 10 → no correction entry, team unchanged
    - `test_blend_medium_drift` — drift 20 (between 10 and 30) → new_elo = old + 20*0.5 = old + 10, reason="blended_50pct"
    - `test_overwrite_large_drift` — drift 50 (> 30) → new_elo = eloratings_value exactly (overwrite), reason="overwrite_drift_gt_30"
    - `test_negative_drift_ignore` — drift -8 (< 10, negative) → ignored
    - `test_negative_drift_blend` — drift -25 (abs 25, between 10 and 30) → blended
    - `test_negative_drift_overwrite` — drift -100 (abs 100 > 30) → overwrite
    - `test_edge_tolerance_boundary` — drift exactly 10 → ignored (tolerance is <10, not <=10)
    - `test_edge_blend_boundary_low` — drift exactly 10.1 → blended
    - `test_edge_blend_boundary_high` — drift exactly 30 → overwrite (>30, not >=30)

    **Class `TestValidation`** (tests validate_eloratings_data):
    - `test_valid_data` — 48+ entries all in [1000,2500] → (True, [])
    - `test_too_few_teams` — < 48 entries → (False, [...])
    - `test_rating_out_of_range_low` — rating 900 → (False, ["Rating 900 for XX out of range"])
    - `test_rating_out_of_range_high` — rating 3000 → (False, ["Rating 3000 for XX out of range"])
    - `test_multiple_validation_errors` — Both too few teams AND out-of-range → multiple messages

    - `test_red_72_to_168h` — 100 hours → (3, "red")
    - `test_critical_above_168h` — 200 hours → (4, "critical")
    - `test_exact_boundaries` — exactly at threshold values (24, 48, 72, 168) — must be AT the threshold to trigger next level (e.g., 24.0 → level 1, not level 0)
