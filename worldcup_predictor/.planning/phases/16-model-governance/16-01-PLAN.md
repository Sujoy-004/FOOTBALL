<objective>
Build the version tracking foundation: governance constants, state.py persistence for versions+snapshots+backtest reports, and governance.py with pure-data version computation functions.

</objective>

    1. `_compute_data_version(old_versions: dict, prev_history: list[dict], new_history: list[dict]) -> str`
       - Implements D-02: increment only if (A) new match_id in new_history not in prev_history, OR (B) existing entry gained a new signal key.
       - Parse current data_version number (strip "D" prefix), increment by 1, return "D{N}".
       - Check condition A: convert prev_history match_ids to set, convert new_history match_ids to set. If new - prev is non-empty, increment.

       - Edge: if old_versions has no data_version or it's "D0", start at "D1" for first increment.
