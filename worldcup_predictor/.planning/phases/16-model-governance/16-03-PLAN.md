    ```python
    def backtest_tournament(
        tournament_matches: list[dict],
        teams: dict[str, dict],
        tournament_name: str = "",
    ) -> dict:
        """Replay a historical tournament through the Elo pipeline.
        
        Takes a list of historical match dicts (with team_a, team_b, actual),
        replays them chronologically through expected_score() and apply_elo_update(),
        computes per-signal metrics and a winner prediction.
        
        Args:
            tournament_matches: List of dicts with team_a, team_b, actual, signals.elo.
            teams: Team data dict (deep-copied for replay — original unchanged).
            tournament_name: Label for the report (e.g., "2018").
        
        Returns:
            Per-tournament report dict:
            {
                "tournament": str,
                "n_matches": int,
                "per_signal": {
                    "elo": {"brier": float, "log_loss": float, "ece": float, "n": int},
                    ...
                },
                "winner_prediction": {"predicted": str, "actual": str, "correct": bool},
                "signal_ranking": ["elo", ...],
            }
        """
    ```

    Implementation:
    1. Deep-copy teams (Pitfall 6: copy.deepcopy(teams)).
    2. Sort matches chronologically (they should already be, but guard).
    3. Iterate matches: compute expected_score for team_a, collect (prediction, actual) pairs, apply_elo_update.
    4. Compute metrics via compute_metrics().
    5. Compute calibration via calibration_curve().
    6. Determine "winner prediction": the team with highest initial Elo rating in the tournament.
       - At tournament start (initial replay_teams state before any matches), find team with max Elo.
       - This is a simplification — full simulation-based winner prediction can be added later.
       - Compare predicted winner vs actual tournament winner.
    7. Return report matching per-tournament format.

    ```python
        replays through backtest_tournament(), collects per-tournament reports,
        produces aggregate report, saves to eval_backtest_report.json.
        
        Args:
            teams: Current team data (deep-copied for replay).
            historical_data_dir: Path to data/historical/ directory.

        Returns:
            Aggregate report dict with per-tournament results, or None if no data.
        """
    ```

    2. For each tournament in `constants.GOV_BACKTEST_TOURNAMENTS` (["2018", "2022"]):

       b. Call `backtest_tournament(matches, teams, tournament_name=tournament)`.
