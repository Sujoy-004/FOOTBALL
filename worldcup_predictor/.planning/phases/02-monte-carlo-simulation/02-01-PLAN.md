  <action>
    Create `src/simulation.py` implementing the Monte Carlo simulation engine. Follow existing module patterns (`elo.py`, `state.py`): pure functional style, no classes, no I/O.

    **Module structure (per D-05, D-02):**

    - `ROUND_ORDER = ["R16", "QF", "SF", "FINAL"]` — module-level constant (per D-01)
    - `ROUND_KEYS = {"QF": "qf", "SF": "sf", "FINAL": "final"}` — maps round names to output keys (per D-07). R16 is NOT in output keys (teams start in R16).
    
    - `_build_round_map(bracket: list[dict]) -> dict[str, list[dict]]` — groups matches by round, sorts each group by match_id. One-time pre-computation outside the 50K loop. Per Pattern 1 in RESEARCH.md.

    - `_build_match_index(bracket: list[dict]) -> dict[str, dict]` — `{match_id: match}` lookup dict. One-time pre-computation.

    - Four private round simulation functions (per D-02):
      - `_simulate_r16(round_map, played, winner_progression, _rand, _exp, elo_ratings)`
      - `_simulate_qf(round_map, played, winner_progression, _rand, _exp, elo_ratings)`
      - `_simulate_sf(round_map, played, winner_progression, _rand, _exp, elo_ratings)`
      - `_simulate_final(round_map, played, winner_progression, _rand, _exp, elo_ratings)`
      
      All four have the same signature. Each iterates over its round's matches in the pre-computed round_map.
      
      Per-iteration logic per match:
      1. Check `if mid in played` — if yes, read `winner` from `played[mid]`, store in winner_progression (per D-15), `continue`
      2. Determine participants:
         - R16: read `match["team_a"]`, `match["team_b"]` directly (always non-null per bracket structure)
         - Post-R16: read `match["source_matches"]` array, resolve participants from `winner_progression[src[0]]` and `winner_progression[src[1]]` (per Pitfall 1 guidance)
      3. Compute `p_a = _exp(elo_ratings[team_a], elo_ratings[team_b])`
      4. `winner = team_a if _rand() < p_a else team_b` (per research recommendation: `random.random() < p` at ~70ns, not `random.choice()` at ~900ns)
      5. `winner_progression[mid] = winner` (per D-15)

    - `run_simulation(teams, bracket, played, iterations=50000, seed=None) -> dict[str, dict[str, float]]`
      
      Per D-03 exact signature. Steps:
      
      1. **Seed** (per D-09): `if seed is not None: random.seed(seed)` — call exactly once at function start, NOT inside the loop (per Pitfall 5).
      
      2. **Pre-compute** (per Pattern 1 + Performance Checklist items 1-4):
         - `round_map = _build_round_map(bracket)`
         - `match_index = _build_match_index(bracket)` (not used by round functions directly — available if needed)
         - `elo_ratings = {name: data["elo"] for name, data in teams.items()}` — flat dict for fast lookup (Performance Checklist item 4)
      
      3. **Aggregation counters** (per D-14): Use `defaultdict(lambda: defaultdict(int))` for per-team, per-round advancement counts. This gives `counts[team_name][round_key] = count` pattern. (Recommended over sets in RESEARCH.md Pattern 3)
      
      4. **Local bindings for hot path** (per Pattern 2 + Performance Checklist item 2):
         ```python
         _rand = random.random
         _exp = expected_score
         _played_get = played.get
         ```
      
      5. **Main loop**: `for _ in range(iterations):` — per D-14, create fresh `winner_progression: dict[str, str] = {}` each iteration. Call the four round functions in order. BE CAREFUL: pass `_rand` and `_exp` locals — the round functions must use these, not access `random` or `elo.expected_score` via global lookup.
      
      6. **Per-iteration tallying** (after all rounds complete in this iteration):
         - For each round_name in ROUND_ORDER:
           - For each match in round_map[round_name]:
             - For each src in match["source_matches"]:
               - If src in winner_progression:
                 - team = winner_progression[src]
                 - Round key: ROUND_KEYS[round_name] (e.g., "QF" → "qf")
                 - counts[team][round_key] += 1
         - Champion: if `"FINAL" in winner_progression`: `counts[winner_progression["FINAL"]]["champion"] += 1`

         Note: R16 matches have source_matches=None, so the loop correctly skips them (teams don't "advance to" R16 — they start there).
      
      7. **Convert counts to probabilities**: For each team in `teams`, divide each round_key count by `iterations`. Return dict. Per D-06 exact shape. Per D-10: no metadata in return.

    **Edge cases to handle (agent's discretion per D-16):**
    - All matches already played: every iteration produces identical winner_progression, probabilities are 0.0 or 1.0. Works naturally.
    - Post-R16 match where source match winner isn't in winner_progression: should not happen in a valid bracket (validated by Phase 1), but guard with KeyError if desired.
    - Empty bracket: return `{team: {"qf": 0, "sf": 0, "final": 0, "champion": 0} for team in teams}`.

    **Performance-critical implementation details:**
    - `_rand` and `_exp` are passed as arguments to round functions (not accessed as globals) — this avoids global name lookup overhead in the hot path
    - `played` is checked via `mid in played` (not `played.get(mid)`) per Performance Checklist item 5
    - Winner_progression is a plain `dict`, not `defaultdict` — we control all insertions explicitly
    - No deep copy of anything inside the loop — per D-14 and Anti-Pattern 1
