## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Agent (fetcher.py) → Football-Data.org API | Untrusted external API response crosses into the prediction engine |
| OS environment → Agent | API key from env var (FOOTBALL_API_KEY) enters process memory |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | fetcher.py logging | mitigate | Never log the full API key value. Log `[REDACTED]` or truncate. Only log the API key status (present/missing/invalid). |
| T-03-02 | Tampering | fetch_raw_matches → response parsing | mitigate | Wrap `resp.json()` in try/except for `json.JSONDecodeError` per D-15. Malformed responses return [] instead of propagating exception. |
| T-03-03 | Spoofing | API response → match data | accept | Team-name matching via aliases (D-01) is sufficient for MVP. No API response signing — accept risk for tournament data. |
| T-03-04 | Denial of Service | fetch_raw_matches retry loop | mitigate | 3-retry cap prevents infinite loops. Exponential backoff (1s/2s/4s) avoids hammering API. Timeout from API_TIMEOUT (10s, D-17) prevents hanging requests. |
| T-03-SC | Tampering | package installs | mitigate | No new packages in this plan. All imports are stdlib (`json`, `time`, `logging`, `os`, `typing`) + already-installed `requests`. No legitimacy audit needed. |
</threat_model>
