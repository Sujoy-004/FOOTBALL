<action>
Create `src/enrichment.py` with:

1. `_STATS_FIELD_MAP` dict — internal name → tuple of BSD field paths to try:
   - `yellow_cards_home`: `("yellow_cards_home",)` — probe confirmed `live_stats.home.yellow_cards`
   - `yellow_cards_away`: `("yellow_cards_away",)`
   - `red_cards_home`: `("red_cards_home",)`
   - `red_cards_away`: `("red_cards_away",)`
   - `shots_on_target_home`: `("shots_on_target_home",)`
   - `shots_on_target_away`: `("shots_on_target_away",)`
   - `possession_home`: `("possession_home",)` — BSD field is `ball_possession`
   - `possession_away`: `("possession_away",)`

2. `_resolve_field(live_stats_side: dict, field_names: tuple[str, ...])` helper — tries each field name in priority order, returns typed value or None. Integer fields (cards, shots) use `int()`, int fields (possession 0-100) use `int()`.

3. `extract_stats(raw_event: dict) -> dict | None`:
   - If `raw_event.get("live_stats")` is None or not a dict → return None
   - Extract `live_stats.home` and `live_stats.away` dicts
   - For each internal field name in `_STATS_FIELD_MAP`:
     - Extract BSD field name from the map (tuple[0])
     - Extract _home variant from `live_stats.home[bsd_name]`
     - Extract _away variant from `live_stats.away[bsd_name]`
     - If a value is found (not None), add it to the result dict
   - Return result dict (may be empty if no stats acquired — per D-07 sparse-fields convention)

4. `_CONTEXT_FIELD_MAP` dict:
   - `venue`: `("name",)` — extracted from `raw_event.get("venue", {})`
   - `referee`: `("name",)` — extracted from `raw_event.get("referee", {})`

5. `_extract_nested(source: dict, target_key: str, field_names: tuple[str, ...])` — extracts value from a nested dict using field name fallback chain.

6. `extract_context(raw_event: dict) -> dict | None`:
   - Extract venue dict from `raw_event.get("venue")`, get `"name"` → `result["venue"]`
   - Extract referee dict from `raw_event.get("referee")`, get `"name"` → `result["referee"]`
   - If neither field resolved (both source dicts are None/missing), return None
   - Return partial dict if only one resolved (per D-07)

2. **BSD field path mapping** for `_STATS_FIELD_MAP`. The probe (RESPONSE.md:C) confirmed:
   - `live_stats.home.yellow_cards` is the exact BSD path for yellow cards
   - `live_stats.home.red_cards` for red cards
   - `live_stats.home.shots_on_target` for shots on target
   - `live_stats.home.ball_possession` for possession (note: BSD calls it `ball_possession`, internal name is `possession_home`)

   Since all P0 stat fields live under `live_stats.home.*` / `live_stats.away.*`, the `_STATS_FIELD_MAP` stores only the leaf name (e.g. `"yellow_cards"`). The `extract_stats()` function navigates `live_stats.home`/`live_stats.away` before applying the leaf lookup.

3. No available/reason flags inside stats or context per D-08.
</action>
