  **Function 2: `_tiebreak_group(team_data_list: list[dict], results: dict[str, dict], depth: int = 0) -> list[dict]`**
  - Recursively sorts a list of team standings dicts using the 7-step tiebreaker chain.
  - Per D-13: exact step order (H2H-first for 2026):
    1. Points in matches between tied teams (H2H points)
    2. Goal difference in H2H matches (H2H GD)
    3. Goals scored in H2H matches (H2H GS)
    4. Goal difference in all group matches (Overall GD)
    5. Goals scored in all group matches (Overall GS)
    6. Fair play conduct score (lower = better, per D-16)
    7. FIFA ranking (lower rank number = better, per D-16)
  - Per D-14: recursive narrowing:
    - Sort by points. If a single team is separated (has unique points), assign its position, recurse on remaining.
    - If 2+ teams have same points: start tiebreaker chain step by step.
    - At each step, if a subset is isolated (e.g., H2H points separate 1 from 2 others), assign the separated team(s) and recurse on the remaining tied cluster from STEP 1.
    - If a step doesn't resolve (all still tied), continue to next step.
    - If all 7 steps fail to separate, break ties by group letter order (arbitrary but deterministic).
  - `team_data_list` entries are dicts with keys: `team`, `pts`, `gd`, `gs`, `conduct_score`, `fifa_rank`. (Note: `conduct_score` is already computed total for all matches; `fifa_rank` needs to be resolved from teams.json — see below.)
  - `results` param: the match results dict for this group (keyed by match_id) — used for H2H recomputation among tied subsets.
  - `depth` param: guards against infinite recursion. If depth > 10, raise ValueError.

  **The H2H computation (_head_to_head_points, _head_to_head_gd, _head_to_head_gs):**
  For a given subset of tied teams, scan all matches in the group results dict. Consider only matches where BOTH teams are in the `tied_teams` set. Compute:
  - H2H points: standard 3/1/0 per match between tied teams
  - H2H GD: sum of (score_a - score_b) for matches between tied teams
  - H2H GS: sum of goals scored by each team in matches between tied teams

  **Function 3: `compute_standings(results: dict[str, dict[str, dict]]) -> dict[str, list[dict]]`**
  - Signature per ARCHITECTURE §5.2

       - **GS**: sum of goals scored (score_a if team_a, score_b if team_b)
        - **Conduct score**: sum of `_compute_conduct_score(yellow_cards, red_cards)` across the team's matches (positive penalty points — lower = better per RESPONSE.md Clarification 1)
        - **FIFA ranking**: Since `teams.json` doesn't store FIFA rank, use **Elo as a proxy** per RESPONSE.md Clarification 2. Rationale: Elo already exists for every team, correlates with FIFA rank, and higher Elo = better team = wins tiebreak (same semantics as lower FIFA rank number). **(Implementation):** Sort key uses `-elo` (descending — higher Elo = better).     2. Sort initial by points descending.
    3. If all 4 teams have unique points → simple sort, assign positions.
    4. If tie exists → extract tied subset → call `_tiebreak_group(tied_teams, results, depth=0)`.
    5. Merge sorted subsets into final list, assign positions 1-4.
  - Returns `{group_letter: [{team: str, pts: int, gd: int, gs: int, yellow_cards: int, red_cards: int, conduct_score: int, position: int}, ...]}`
  - Also return each team's total yellow_cards and red_cards across all group matches (summed from results).

  **CONCRETE VALUES TO USE:**
  - Point system: WIN=3, DRAW=1, LOSS=0
  - Sort direction for H2H points/GD/GS, overall GD/GS: descending (higher number = better, per D-16)
  - Sort direction for conduct_score: ascending (lower positive penalty points = better, per D-16 + RESPONSE.md Clarification 1)
  - Sort direction for fifa_rank: ascending (lower rank number = better, per D-16)
