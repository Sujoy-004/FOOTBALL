  **Function: `resolve_r32_matchups(advancers: dict[str, dict[int, str]], standings: dict[str, list[dict]], third_ranked: list[dict], annex_c: dict) -> dict[str, dict]`**
  - Inputs:
    - `advancers`: output of `select_advancers()` — `{group: {1: winner, 2: runner_up, 3: third_team_or_none}}`
    - `standings`: output of `compute_standings()` — needed for teams' full data
    - `third_ranked`: output of `rank_third_placed()` — sorted list of third-placed teams (top 8 advance)

  - Returns: `{match_id: {team_a: str, team_b: str, match_id: str}}` for all 16 R32 matches.
  
  **Algorithm per ARCHITECTURE §3.1:**

  1. **Select advancing third-placed:** `top8 = third_ranked[:8]` → `advancing_groups = set(t.group for t in top8)`

  2. **Build Annex C key:** `key = ",".join(sorted(advancing_groups))`
     - **[CRITICAL — Pitfall 2 guard]** Keys must be sorted alphabetically ascending, comma-separated. This matches the key format in `annex_c.json`.

  3. **Look up Annex C assignment:**
     - `assignment = annex_c.get(key)` — if key not found, raise `ValueError(f"Annex C key not found: {key}. This is a data integrity error — all 495 combinations should be present.")`
     - Also strip `_meta` key from annex_c before lookup: `clean_annex = {k: v for k, v in annex_c.items() if k != "_meta"}`

  4. **Resolve the 8 "winner vs third" R32 matches:**
     - Per FEATURES.md §4 R32 structure, the 8 matches with `annex_c_third` slots are:
       - M74: Winner E vs 3rd(E) — but "3rd(E)" means the third-place team assigned to face Winner E per Annex C
       - M77: Winner I vs 3rd(I)
       - M79: Winner A vs 3rd(A)
       - M80: Winner L vs 3rd(L)
       - M81: Winner D vs 3rd(D)
       - M82: Winner G vs 3rd(G)
       - M85: Winner B vs 3rd(B)
       - M87: Winner K vs 3rd(K)
     - For each match: look up the assignment entry matching the group_winner (e.g., "1E" for Winner E)
       - `third_group = assignment["1E"]` → e.g., "3C" → extract "C" → third-placed team from Group C
       - `third_team = advancers[third_group][3]` — the third-placed team from that group
       - `winner = advancers[winner_group][1]` — the winner of the winner_group
     - Per D-15 and FEATURES.md §3: The Annex C assignment determines WHICH third-place team goes to WHICH winner slot.

  5. **Resolve the 8 "fixed" R32 matches** (group_position only, per FEATURES.md §4):

     - M75: F1 vs C2 (winner of F vs runner-up of C)

     - Resolve via `advancers[group][position]` where position=1 for winner, 2 for runner-up.
