World Cup 2026 Dynamic Predictor

A live, self-updating tournament predictor for the 2026 FIFA World Cup

In plain English: This tool watches World Cup matches as they happen, updates its predictions based on the results, and shows you who has the best chance of winning the tournament — all from your terminal.

What does it do?

Every time a World Cup match finishes, this tool:

  1. Picks up the result — It polls a live sports data API every 60 seconds for new finished matches.
  2. Updates team ratings — It uses the Elo rating system (the same math behind chess rankings) to adjust each team's strength based on the result. A big win against a strong opponent moves a team up more than a narrow win against a weak one.
  3. Reruns the simulation — It simulates the entire tournament 50,000 times to figure out each team's chances of reaching the quarter-finals, semi-finals, final, and winning the whole thing.
  4. Shows you the numbers — Every cycle, it prints updated championship probabilities and group standings to your terminal.

What can you do with it day to day?

Morning check (2 minutes)

Run python main.py --once. See which new matches were ingested overnight, check updated probabilities, and see if any team made a big jump or drop.

During match days

Start continuous polling with python main.py. Leave it running in a terminal window. Each time a match ends, you'll see an alert block with the score, the winner, and the Elo changes.

Compare predictions to reality

Use the daily validation report template to track how well the model is calibrated. When Spain (23% champion) wins, did the probability move the right amount? When Haiti (0.0% champion) loses, did the model correctly indicate they were unlikely?

Track the bubble

The third-place advancement bubble is the most exciting race. The tool shows you exactly which team is 8th (advancing) and 9th (out), and the cutoff margin. Watch this change after every match.

Post-match analysis

After a surprise result — say, South Korea beats Spain — run the update and watch how Spain's 23% champion probability shifts. Is the model overreacting or underreacting? This tells you if the Elo K-factor is tuned right.

What the project does NOT do

  • ✗ It does not predict individual match scores (only tournament progression probabilities)
  • ✗ It does not handle draws (matches ending in a draw are skipped — no Elo update, no impact)
  • ✗ It does not have a web UI — it's a terminal application
  • ✗ It does not save probability history (probs are computed fresh each run)
  • ✗ It does not automatically detect penalty shootout winners (handled as regular wins)

Honest assessment

✓ What works well: Live ingestion from the BSD API is reliable. The Elo update system correctly applies rating changes after each match. Restarting the tool doesn't duplicate data. The 50K Monte Carlo simulation produces stable, interpretable probabilities. Group standings are displayed clearly with third-place bubble tracking.

✗ What needs attention: One test fails because Elo values can be float (1299.0) instead of int (1299) after dynamic updates. This is a minor code-test mismatch. The probabilities are not persisted to disk between runs (computed fresh each simulation). Draws are intentionally skipped — this means the model doesn't capture information from drawn matches, which is a deliberate design tradeoff.

✗ What to watch: When the knockout stage starts (R32 on June 26), the bracket resolution logic will be exercised for the first time. This is the most complex part of the simulation and should be monitored closely.

Bottom line: The tool is ready for daily use. Run it once a day to track shifting probabilities. The predictions are as good as the underlying Elo model and Monte Carlo simulation — which is to say, better than guessing, but no substitute for watching the actual tournament.
