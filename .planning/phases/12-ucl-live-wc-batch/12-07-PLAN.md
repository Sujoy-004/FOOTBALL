  ```python
  def _parse_what_if_file(what_if_path: str, teams: dict) -> dict:
      """Parse and validate --what-if JSON override file.
      
      Expected JSON structure:
      {
          "elo_changes": {"Brazil": 2200.0, "Argentina": 2100.0},
          "blend_weights": {"elo": 0.4, "odds": 0.3, "catboost": 0.3},
          "xg_overrides": {"BRA-ARG": {"home_xg": 1.8, "away_xg": 1.2}},
          "calibration_temperature": 1.2
      }
      
      Returns:
          Dict of validated overrides, or raises ValueError on invalid content.
      """
  ```

  **Validation rules:**
  - `elo_changes` (optional): dict[str, float]. Each team must exist in `teams`. Each value must be > 0.
  - `blend_weights` (optional): dict[str, float]. Keys must match known signal names. Must not sum to 0.
  - `xg_overrides` (optional): dict[str, dict]. Inner dicts must have `home_xg` and `away_xg` keys, each > 0.
  - `calibration_temperature` (optional): float > 0.

  **Edge cases:**
  - Empty file → error
  - Unknown override keys → error listing known keys
  - Team in elo_changes not in teams dict → error: "Unknown team '{name}'"
  - Negative Elo value → error
  </action>

  ```python
  def _run_counterfactual(
      baseline_teams: dict,
      baseline_groups: dict,
      baseline_bracket: list[dict],
      annex_c: dict,
      played: dict,
      played_groups: dict,
      overrides: dict,
      seed: int,
      iterations: int,
  ) -> tuple[dict, dict]:
      """Run counterfactual simulation with overrides applied.
      
      Args:
          baseline_teams: original teams dict (deep-copied before modification)
          overrides: validated overrides dict from _parse_what_if_file()
          seed: base seed (counterfactual uses seed + 1)
          iterations: simulation iteration count
          
      Returns:
          (cf_result, change_descriptions)
          cf_result: run_full_simulation() output
          change_descriptions: list of human-readable strings describing each change
      """
  ```

  **Logic:**
  1. Deep-copy all mutable data structures (`copy.deepcopy`) — baseline_teams, baseline_groups, baseline_bracket.
  2. Apply `elo_changes`: `teams_copy[team]["elo"] = new_value`.
  3. Build `blend_params` from overrides:
     - If `blend_weights` present: `blend_params = {"calibrated": False, "weights": overrides["blend_weights"]}`
     - If `calibration_temperature` present: add to blend_params.
  4. Build `xg_overrides` from overrides if present.
  5. Run `run_full_simulation()` with modified parameters, `seed=seed + 1`.
  6. Build change descriptions: `["Brazil.elo: 1850 → 2200 (+350)"]`.
  7. Return `(cf_result, descriptions)`.

  Add `print_what_if_comparison()` function to `competitions/worldcup/src/output.py`:

  ```python
  def print_what_if_comparison(
      baseline_result: dict,
      counterfactual_result: dict,
      change_descriptions: list[str],
      n_top: int = 10,
  ) -> None:
      """Print side-by-side comparison of baseline vs counterfactual simulation.
      
      Format:
          ==== Counterfactual Analysis ====
          
          Changes:
            Brazil.elo: 1850 → 2200 (+350)
          
          Top-10 Champion Probabilities:
            Team                      Baseline  Counterfactual    Delta
            ───────────────────────────────────────────────────────────
            Brazil                     38.2%          45.1%    +6.9%
            Argentina                  22.5%          20.1%    -2.4%
            ...
      """
  ```

  **Logic:**
  1. Print header: `"==== Counterfactual Analysis ===="`
  2. Print change descriptions (one per line, indented).
  3. Extract champion probabilities: `baseline_result["teams"][team]["champion_prob"] * 100`.
  4. Sort teams by baseline champion prob descending (top `n_top`).
  5. Print table with columns: Team (24 chars), Baseline (9 chars, "XX.X%"), Counterfactual (14 chars, "XX.X%"), Delta (7 chars, "+XX.X%").
  6. Delta column: `counterfactual_pct - baseline_pct`, formatted with `+`/`-` prefix.
  7. Use existing `_green`/`_red` ANSI wrappers for delta colors.
  8. Only show teams with abs(delta) > 0.1% (clean output).

  **Edge cases:**
  - Counterfactual result has different teams than baseline → use union of keys
  - All deltas < 0.1% → print "No material change detected"
  - `change_descriptions` empty → skip the "Changes:" section
  - Team champion_prob is None → treat as 0.0
  
  **ASCII requirement:** Use ASCII dashes only for separator lines. No Unicode characters.
  </action>

  - Delta values have `+` or `-` prefix
  - Positive deltas use green ANSI; negative use red
  - No material change case prints message (not empty output)
  - All output is pure ASCII
