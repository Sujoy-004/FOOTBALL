  ### Function 1: `apply_elo_update(match: dict, elo_ratings: dict[str, float], elo_applied: list[str]) -> dict | None`

  Apply incremental Elo update for a single finished match.

  **Parameters:**
  - `match: dict` — BSD match event with keys `match_id` (str), `home_team` (str), `away_team` (str), `home_score` (int), `away_score` (int), `winner` (str: "home"/"away"/"draw")
  - `elo_ratings: dict[str, float]` — mutable dict mapping team_name → elo_rating. Modified in-place.
  - `elo_applied: list[str]` — list of match_ids already processed. Checked before applying; appended after.

  **Logic:**
  1. If `match["match_id"]` already in `elo_applied`, return `None` (no-op, restart guard per D-03).
  2. Look up home/away team Elo from `elo_ratings`. If either team missing, log warning and return `None`.
  3. Compute `k_factor = football_core.elo.compute_k_factor("league")` (default).
  4. Compute `expected_score(home_elo, away_elo, home_advantage=club_constants.HOME_ADVANTAGE)`.
  5. Call `update_ratings(home_elo, away_elo, home_score, away_score, k_factor)`.
  6. Update `elo_ratings[home_team]` and `elo_ratings[away_team]` in-place.
  7. Append `match["match_id"]` to `elo_applied`.
  8. Return dict: `{"match_id": ..., "home": {"team": ..., "elo_before": ..., "elo_after": ...}, "away": {...}, "delta_home": ..., "delta_away": ...}`

  **Edge cases:**
  - Match already applied (match_id in elo_applied) → return None silently
  - Team not in elo_ratings → log warning, return None (team not tracked)
  - Home/away scores missing → log error, return None
  - PK Shootout matches (winner field "home"/"away" with draw in regulation): football_core.elo.update_ratings already handles this via score parameters; pass actual goals.

  ### Function 2: `sync_elo_from_clubelo(teams: dict[str, dict], elo_applied: list[str], data_dir: str | None = None) -> list[dict]`

  Periodic external Elo sync from ClubElo (eloratings.net) with graduated correction.

  **Parameters:**
  - `teams: dict[str, dict]` — mutable dict mapping team_name → team_data (contains "elo" key). Modified in-place.
  - `elo_applied: list[str]` — list of match_ids (not modified by this function).
  - `data_dir: str | None` — optional data directory path for cache/fallback.

  **Logic:**
  1. Fetch ClubElo TSV via `football_core.elo_sync.fetch_eloratings_tsv(url=CLUBELO_TSV_URL)`.
  2. If fetch fails (returns None), try loading from cache file and log warning. If cache also missing, return empty list.
  3. Parse TSV via `football_core.elo_sync.parse_eloratings_tsv(tsv_raw)`.
  4. Validate: filter to teams present in `teams` dict (unknown teams are skipped with debug log).
  5. Apply graduated correction via `football_core.elo_sync.apply_graduated_correction(teams, parsed_values)`.
  6. Save updated teams to teams JSON via `football_core.state.save_teams(teams, data_dir=data_dir)`.
  7. Return list of correction dicts (each with team, elo_before, elo_after, drift, reason).

  **ClubElo TSV URL constant:** Define at module level: `CLUBELO_TSV_URL = "https://www.clubelo.com/ClubElo.tsv"`. Per RESEARCH.md open question, this URL may need verification — use a try-except wrapper with logging for non-critical failure. If the TSV format differs from `parse_eloratings_tsv()` expectations, add a `_parse_clubelo_tsv()` fallback parser specifically for club format (columns: rank, club, country, rating, change).

  **Edge cases:**
  - TSV fetch fails (network error) → log warning, try cache, return empty list
  - TSV format mismatch (different columns than world TSV) → catch parse error, log error, return empty list
  - All teams already within drift tolerance → return empty list (no corrections)
  - Some teams drift >30 → partial corrections list with only those teams
  - If `elo_applied` is non-empty (some matches already played), skip ClubElo sync for teams that have recent Elo changes — log info: "N match Elos applied this session — ClubElo sync may overwrite recent updates"
