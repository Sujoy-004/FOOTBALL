  ```python
  def _run_calibrated_validation(
      teams: dict,
      groups: dict,
      bracket: list[dict],
      annex_c: dict,
      played: dict,
      played_groups: dict,
      data_dir: str,
      iterations: int = 50000,
  ) -> dict:
      """Run validation twice: uncalibrated (baseline) and calibrated.
      
      Returns:
          dict with structure:
          {
              "baseline": {"brier": ..., "log_loss": ..., "ece": ..., "trps": ..., "champion_acc": ...},
              "calibrated": {"brier": ..., ...},
              "delta": {"brier": ..., ...},
              "n_matches": int,
              "calibration_available": bool,
          }
      """
  ```

  **Logic:**
  1. **Uncalibrated run:** Run `run_full_simulation()` without calibration params. Extract results.
  2. **Calibrated run:** Load calibration params from `CalibrationPipeline.load(data_dir)`. Run `run_full_simulation()` with calibration applied. If no calibration params found, return `{"calibration_available": False, ...}` — only baseline available.
  3. **Metrics computation:** For both runs, compute Brier score, Log Loss, ECE, TRPS, champion accuracy using `football_core.evaluation` functions against actual match results from `played` + `played_groups`.
  4. **Delta:** `calibrated_value - baseline_value` for each metric.

  ### D. `_print_calibration_comparison()` function

  ```python
  def _print_calibration_comparison(validation_result: dict) -> None:
      """Print before/after calibration comparison table.
      
      Format:
          ==== Calibration Validation ====
          
          Metric                Before    After    Delta
          ─────────────────────────────────────────────
          Brier                 0.231    0.218   -0.013
          Log Loss              0.712    0.695   -0.017
          ECE                   0.084    0.062   -0.022
          TRPS                  0.312    0.298   -0.014
          Champion Accuracy     68.2%    71.5%   +3.3%
          
          Calibration improved 3 of 5 metrics.
  ```

  **Logic:**
  - Green for improvement (negative delta for error metrics, positive for accuracy)
  - Red for degradation
  - Summary line: "Calibration improved X of 5 metrics" or "Calibration degraded X of 5 metrics"
  - If no calibration data: "No calibration data found. Run normal polling mode first to build calibration params."
  - If zero played matches: "No played matches to validate against"

  ### B. `_parse_weights()` function

  ```python
  def _parse_weights(weights_str: str, known_signals: list[str]) -> dict[str, float]:
      """Parse K=V,K=V weight string, validate, normalize to sum 1.0.
      
      Args:
          weights_str: "elo=0.4,odds=0.3,catboost=0.3"
          known_signals: list of known signal names for validation
          
      Returns:
          {signal_name: normalized_weight} summing to 1.0
          
      Raises:
          ValueError on invalid format, unknown signals, all-zero weights
      """
  ```

  **Validation:**
  - Split by comma, then by "=". Each pair must have exactly 2 parts.
  - Values must be parseable as floats.
  - All keys must be in `known_signals` list.
  - Values must be >= 0.
  - Sum must be > 0 (not all zeros).
  - Normalize to sum 1.0 (divide each by total).
  - If sum != 1.0 before normalization, print note: "Weights normalized to sum 1.0"

  **Validation errors:**
  - Unknown signal → "Unknown signal 'xyz'. Available: elo, odds, catboost, ..."
  - Non-numeric value → "Invalid weight value '{val}' for signal '{key}'"
  - All-zero weights → "Weights must sum to > 0"
  - Single key only → that key gets weight 1.0 (valid edge case)

  ### C. Wire into _run_batch_mode()

  In `_run_batch_mode()`, before calling `run_full_simulation()`:
  ```python
  blend_params = {}
  if args.weights:
      known = list(_gather_signal_data(...)[0].get("signals", {}).keys()) if signal_data else []
      # Or known signals from prediction_history/config
      static_weights = _parse_weights(args.weights, known_signals)
      blend_params = {"calibrated": False, "weights": static_weights}

  Then pass `blend_params` to `run_full_simulation()`:

  )
  ```

  The `blend_params` with `"calibrated": False` and `"weights": {...}` tells the simulation engine to use static weights directly, skipping Brier optimization and calibration (per WC-BATCH-12).
