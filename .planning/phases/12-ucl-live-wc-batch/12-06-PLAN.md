<objective>
Add offline simulation mode to WorldCup CLI — `--simulate` flag, `--iterations` override, and mode routing.

**Purpose:** WorldCup currently requires live polling with BSD API access. This plan adds a self-contained `--simulate` mode that loads data from local files and runs a single simulation — no API needed. Enables reproducible research, batch analysis, and offline use.

**Output:**
- Modified `competitions/worldcup/main.py` with `--simulate` flag, `--iterations N`/`-n` flag
- Mode routing: batch vs polling
- Staleness warning for data files >24h old
</objective>

  ### A. Mode routing in main()

  After arg parsing and before entering the existing polling loop, add:


  ### B. `_run_batch_mode()` function

  ```python
  def _run_batch_mode(args: argparse.Namespace, data_dir: str) -> None:
      """Run offline simulation --simulate mode: load data files, run MC, display results."""
  ```

  **Function body:**

  1. **Load data files** using existing state loaders:
     - `teams = load_teams(data_dir)`
     - `groups = load_groups(data_dir)`
     - `bracket = load_bracket(data_dir)`
     - `annex_c = load_annex_c(data_dir)`
     - `played = load_played(data_dir)`
     - `played_groups = load_played_groups(data_dir)`

  2. **Staleness check** (per Pitfall 2 in RESEARCH.md):
     - For each data file, check `os.path.getmtime()`.
     - If any file > 24h old, print warning: `"Warning: {filename} last modified {date}"`
     - Check specifically: `teams.json` (Elo ratings may be stale).

  3. **Build Elo dict** from `teams[team_name]["elo"]`.

  4. **Extract Elo seed** from args: `seed = args.seed or 42`.

  5. **Set iteration count:** `iterations = args.iterations or 50000`.

  6. **Run full simulation:**
     ```python
     result = run_full_simulation(
         teams, groups, bracket, annex_c, played, played_groups,
         seed=seed, iterations=iterations,
     )
     ```

  7. **Display results:**
     - Call existing `print_probability_table(result)` (from `output.py`).
     - Print simulation metadata: seed, iterations, duration.

  8. **Handle missing data files:**
     - If any required file is missing, print clear error listing which files are missing: `"Error: Missing required data files: teams.json, groups.json"`

  **Edge cases:**
  - All data files missing → list all missing, exit 1
  - teams.json has no "elo" key for some teams → skip that team with warning
  - run_full_simulation returns empty result → print "Simulation produced no results" warning
  - `--simulate` with `--once` → --simulate takes precedence (--once is for polling loop)
  </action>
