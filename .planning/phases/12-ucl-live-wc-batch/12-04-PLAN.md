  Add five new functions to `competitions/ucl/display.py`:

  ### 1. `print_delta(probs: dict[str, float], prev_probs: dict[str, float] | None) -> None`

  Print compact delta column comparing current vs previous probabilities.

  **Logic:**
  - If `prev_probs` is None (first cycle), print nothing and return.
  - For each team in `probs`, compute `delta = probs[team] - prev_probs.get(team, 0.0)`.
  - If `abs(delta) < 0.5` (percentage points), skip (no noise for tiny changes).
  - Print: `Team   Current   Previous   Delta` table.
  - Use `+`/`-` prefix (ASCII per Phase 11 D-04 — no Unicode arrows). Use ANSI green (via `_green`) for positive, red (`_red`) for negative.
  - Show delta as percentage with 1 decimal: `+3.2%`, `-1.5%`.
  - Teams sorted by abs(delta) descending.

  **Edge cases:**
  - Team in `probs` but not in `prev_probs` → delta = prob (treated as first appearance)
  - All deltas < 0.5% → print nothing (clean output)
  - `prev_probs` is empty dict → same as None (no-op)

  ### 2. `print_heartbeat(timestamp: str, next_poll_seconds: int, new_matches: int) -> None`

  Print a timestamped status line for idle cycles.

  **Format:** `[14:32:15] No new matches — next poll in 60s`
  Or with new matches: `[14:32:15] Ingested 3 new match(es) — next poll in 60s`

  **Logic:**
  - `timestamp` is a time string like "14:32:15" (formatted by caller).
  - `next_poll_seconds` is the remaining interval.
  - `new_matches` is count of new matches ingested this cycle.
  - Single ASCII line, no ANSI codes needed.

  ### 3. `print_match_alert(match: dict) -> None`

  Print a single-line alert when a new match result is ingested.

  **Format:** `→ Team A 2-1 Team B (match_id)`

  **Logic:**
  - Extract `home_team`, `away_team`, `home_score`, `away_score` from match dict.
  - Print using ASCII characters only (right arrow `→` is allowed — it's ASCII 26, but per Phase 11 D-04 ASCII convention, use `>` instead to be safe). Use `>` for separator.
  - Only called for new matches (not re-displayed on every poll).

  ### 4. `print_elo_changes(updates: list[dict]) -> None`

  Print Elo changes after match processing.

  **Format (for each updated team):** `  Team A: 1845 → 1858 (+13)`
  Or multiple: `  Team A: 1845 → 1858 (+13), Team B: 1720 → 1709 (-11)`

  **Logic:**
  - `updates` is a list of dicts from `apply_elo_update()` return values.
  - For each update, extract home and away team changes.
  - Print comma-separated on one line for compactness.
  - If `updates` is empty or None, print nothing.

  ### 5. `print_simulation_duration(seconds: float) -> None`

  Print MC simulation duration, right-aligned.

  **Format:** `Simulation: 8.1s`
  If > 60s: `Simulation: 1m 23.4s`

  **Logic:**
  - `seconds` is float duration.
  - < 60s: show with 1 decimal: `Simulation: 8.1s`
  - >= 60s: show as minutes + seconds: `Simulation: 1m 23.4s`
  - Use ANSI dim for secondary text.

  **All functions must be pure ASCII** — no Unicode arrows, no special chars. ANSI escape codes permitted.

  ### A. `_merge_signals_into_history()` function

  ```python
  def _merge_signals_into_history(
      matches: list[dict],
      signal_caches: dict[str, dict],
      prediction_history: list[dict],
  ) -> list[dict]:
      """Merge signal cache data into prediction_history for each match.
      
      Args:
          matches: List of BSD match event dicts (finished matches)
          signal_caches: {cache_key: {match_id: signal_data}} from cache files
          prediction_history: Existing history list (mutated in-place for new entries)
          
      Returns:
          Updated prediction_history with new entries appended
      """
  ```

  **Logic:**
  1. For each match in `matches`, check if an entry already exists in `prediction_history` (by match_id).
  2. If not, create a new entry: `{match_id, home_team, away_team, home_score, away_score, signals: {}, timestamp}`.
  3. Fill `signals` from cache data: look up match_id in each signal cache. Available signal keys: `odds` (from odds_cache), `catboost` (from catboost_cache), `elo` (computed on-the-fly).
  4. Append new entries to `prediction_history`.
  5. Return updated `prediction_history`.

  **Edge cases:**
  - Match already in prediction_history → skip (don't duplicate)
  - Signal cache missing for a match → leave signal slot empty (key exists with None value)
  - All signals missing for all matches → history entries created with empty signals dicts

  ### B. Signal cache refresh in _run_iteration()

  In `_run_iteration()`, before the MC simulation step, add cache refresh logic:

  ```python
  # Load cache TTL config
  cache_ttls = _load_cache_ttls(data_dir)
  
  # Refresh signal caches (odds, catboost)
  for cache_key in ["odds", "catboost"]:
      cache = load_ucl_cache(cache_key, data_dir)
      ttl = cache_ttls.get(cache_key, 12 if cache_key == "odds" else 24)
      
      needs_refresh = not ucl_is_cache_valid(cache, ttl)
      # Also refresh if new matches were ingested (D-04: TTL OR new match event)
      if new_matches_count > 0:
          needs_refresh = True
      
      if needs_refresh:
          try:
              new_cache = _fetch_signal_cache(cache_key, ...)
              save_ucl_cache(cache_key, new_cache, data_dir, ttl_hours=ttl)
          except Exception as e:
              logging.warning(f"Failed to refresh {cache_key} cache: {e}")
  ```

  ### C. `_load_cache_ttls()` helper

  ```python
  def _load_cache_ttls(data_dir: str) -> dict[str, int]:
      """Load per-signal cache TTLs from config/cache_ttls.json.
      
      Returns dict with defaults:
          {"odds": 12, "catboost": 24}
      """
      config_path = os.path.join(data_dir, "..", "config", "cache_ttls.json")
      try:
          with open(config_path) as f:
              return json.load(f)
      except (FileNotFoundError, json.JSONDecodeError):
          return {"odds": 12, "catboost": 24}
  ```
