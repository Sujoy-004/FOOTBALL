  5. **`fetch_ucl_matches(api_key, aliases, fixtures_schedule) -> list[dict]`:**
     - Build alias lookup from `_build_alias_lookup(aliases, bracket=[])` — empty bracket list because UCL league phase has no bracket.
     - Also register fixture teams: iterate `fixtures_schedule.get("teams", [])`, add `team["name"].strip().lower() -> team["name"]` to alias_lookup for teams missing from BSD data.
     - Get raw data: `fetch_raw_matches(api_key, build_ucl_url(), league_id=UCL_LEAGUE_ID)`.
     - Build fixture lookup dict: iterate `fixtures_schedule.get("matchdays", [])`, for each match in each matchday, add both orientations `(team_a, team_b) -> match_id` and `(team_b, team_a) -> match_id`.
     - For each event in raw: skip if `event.get("status") != "finished"`. Skip unmatchable teams (log warning using exact WC pattern: `logger.warning("Unmatchable teams: home=%r, away=%r", home_name, away_name)`). Skip if no fixture match found (log warning: `logger.warning("No fixture match for %s vs %s", home_norm, away_norm)`).
     - Extract score and determine winner/draw: use exact same pattern as `competitions/worldcup/src/fetcher.py` lines 72-93 (score comparison, NOT BSD `winner` field for draw detection).
     - Build entry dict with keys: `match_id`, `team_a`, `team_b`, `winner`, `is_draw`, `home_score`, `away_score`, `completed_at`.
     - If `event.get("odds_home")`, `event.get("odds_draw")`, `event.get("odds_away")` are all not None: import `remove_vig` from `football_core.predictors.odds` (local import to avoid circular dependency), compute fair odds, store in `entry["odds"]`.
     - Return results list.

  **What to AVOID:**
  - Do NOT include stats/context/preview extraction (WC has enrichment.py, UCL doesn't).
  - Do NOT include bracket matching (UCL league phase has no bracket).
  - Do NOT hardcode the API key anywhere — it's always passed as a parameter.
  - Do NOT call `remove_vig` at module level — import it inside the loop to avoid import-time failures.
  </action>
