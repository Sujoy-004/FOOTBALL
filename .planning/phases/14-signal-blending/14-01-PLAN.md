    - `EPS = 1e-15` (matches evaluation.py log_loss clamping convention)
    - `RIDGE = 1e-6` (ridge regularization for Hessian diagonal — from research Section 1 pitfall 2)
    - `MAX_ITER = 50` (Newton-Raphson convergence guard)
    - `CONV_TOL = 1e-6` (convergence threshold for parameter step size)

    **Private helpers:**

    `_sigmoid(x: float) -> float`: Numerically stable sigmoid. For x < -100 return EPS. For x > 100 return 1-EPS. Otherwise return 1/(1+exp(-x)). This prevents float overflow (research Pitfall 1).

    `_log_odds(p: float) -> float`: Clamp p to [EPS, 1-EPS], return log(p/(1-p)). Clamping prevents ±inf from log(0) or log(1) (research Pitfall 1).

    `_platt_targets(actuals: list[float]) -> list[float]`: Implement Platt's target adjustment from research Section 1 item 4:
      - Count n_pos (actual == 1.0) and n_neg (actual == 0.0)
      - t_pos = (n_pos + 1) / (n_pos + 2) if n_pos > 0 else 0.5
      - t_neg = 1 / (n_neg + 2) if n_neg > 0 else 0.5
      - For each actual: 1.0 → t_pos, 0.0 → t_neg, 0.5 (draws) → 0.5
      This prevents overfitting at probability extremes.

    **Public exports:**

    `calibrate_signal(predictions: list[float], actuals: list[float], threshold: int = COLD_START_THRESHOLD) -> tuple[float, float]`:
      - Per D-02: Fits Platt scaling on log-odds of predictions vs actual outcomes (0/0.5/1)
      - Per D-03: If len(predictions) < threshold, return (1.0, 0.0) for identity calibration
      - Per D-01: Pure Python Newton-Raphson (NOT scipy, NOT sklearn)
      - Algorithm (from research Section 1):
        1. Convert predictions to log-odds via _log_odds()
        2. Compute Platt-adjusted targets via _platt_targets()
        3. Initialize A=0.0, B=0.0 (identity initialization)
        4. Iterate up to MAX_ITER times:
           a. Compute f = [A*xi + B for xi in x], p = [_sigmoid(fi) for fi in f]
           b. Gradient: dA = sum(xi*(pi - ti)), dB = sum(pi - ti)
           c. Hessian with ridge: H_AA = sum(xi*xi*pi*(1-pi)) + RIDGE, H_AB = sum(xi*pi*(1-pi)), H_BB = sum(pi*(1-pi)) + RIDGE
           d. Determinant: det = H_AA*H_BB - H_AB*H_AB; if abs(det) < 1e-12 break
           e. Newton step via Cramer's rule (NOT numpy — per research "Don't Hand-Roll" matrix inversion table):
              dA_step = (H_BB*dA - H_AB*dB)/det, dB_step = (H_AA*dB - H_AB*dA)/det
           f. A -= dA_step, B -= dB_step
           g. If abs(dA_step) < CONV_TOL and abs(dB_step) < CONV_TOL: break
        5. Return (A, B)
      - Per D-22 (graceful degradation): Never raise exceptions. Return (1.0, 0.0) for any edge case (empty input, all-draws, etc.)

    `apply_calibration(p_raw: float, A: float, B: float) -> float`:
      - Per D-02: returns sigmoid(A * log_odds(p_raw) + B)
      - Clamp p_raw to [EPS, 1-EPS] before log-odds transform
      - Clamp result to [0.0, 1.0] and round to 6 decimal places (matching evaluation.py convention)
      - Identity case: (A=1.0, B=0.0) → returns p_raw (identity transform)

    **Part A — Add functions to `src/blender.py` (append after Task 1 functions):**

    `compute_rolling_brier(signal_key: str, window: int = BRIER_WINDOW_SIZE) -> float`:
      - Reads prediction_history via `state.load_prediction_history()` (import state inside function to avoid circular imports)
      - Collects (probability, actual) pairs for signal_key where available=True and probability is not None and actual is not None
      - Takes last `window` entries by list order (history is chronologically ordered per research Assumption A3)
      - Returns mean Brier: sum((p-a)**2)/n if pairs exist; returns 1.0 if no pairs (worst case)
      - Performance note: for N_history ~104 entries and 3 signals, this is ~300 iterations — negligible

    `compute_blend_weights(signal_briers: dict[str, float]) -> dict[str, float]`:
      - Per D-07: for each key, raw_weight = 1.0 / max(brier, 0.05)
      - Floor at 0.05 prevents division by zero (D-07) and extreme weights from near-perfect signals
      - Normalize: divide each raw_weight by sum of all raw_weights
      - Edge case: if sum(raw_weights) == 0, return equal weights for all signals (graceful degradation per D-22)
      - Round output weights to 6 decimal places (matching evaluation.py convention)

    `blend_predictions(signal_preds: dict[str, float], weights: dict[str, float]) -> float`:
      - Only includes signals present in BOTH signal_preds AND weights (signal is available and has a weight)
      - Re-normalizes weights to sum to 1 for the available subset (pitfall 6 from research: blend weights for missing signals)
      - If no signals available: return 0.5 (uniform prior — graceful degradation per D-22)
      - Blended = sum(weight[s] * prob[s] for available signals) / sum(weight[s] for available signals)
      - Round result to 6 decimal places
      - This function is called per-match during simulation blending

    `loo_cv_blended_brier(histories: dict[str, tuple[list[float], list[float]]]) -> float`:
      - Takes aligned per-signal prediction/actual pairs: `{signal_name: (predictions_list, actuals_list)}`
      - All signal lists must be the same length (aligned by match index)
      - Per D-11: Leave-one-out cross-validation:
        For each index i:
          1. For each signal s: train on all indices except i via calibrate_signal(train_preds, train_actuals, threshold=1)
             (use threshold=1 so that even n-1 >= 1 trains; the identity guard applies only for real cold-start)
          2. Compute train Brier for each signal via `compute_metrics(train_preds, train_actuals)["brier"]`
             (import compute_metrics from evaluation inside the function to avoid circular imports)
          3. Compute calibration params (A_s, B_s) from training data
          4. Compute calibrated probability for held-out sample i: apply_calibration(pred_i, A_s, B_s)
          5. Compute blend weights from training Briers via compute_blend_weights(train_briers)
          6. Blend calibrated probs via blend_predictions
          7. Brier error: (p_blended - actual_i) ** 2
        Return mean of all Brier errors.
      - If n < 2 or no signals: return 1.0
      - Performance: for N=30 and 3 signals, ~90 calibrate_signal calls (~8K ops) — negligible per research Section 5

    `compute_poisson_base_rate(match_data_path: str | None = None) -> float`:
      - Per V2-09, D-09, D-10: Compute expected goals per team per match from historical WC data
      - If match_data_path is None or file doesn't exist: return 1.25 (current default from constants.py, per research Option C)
      - If file exists: load JSON data, compute total_goals / total_matches / 2 (goals per team per match)
      - Return float rounded to 4 decimal places
      - This is infrastructure; the data file creation is gated (research Open Question 1). The function gracefully falls back.
