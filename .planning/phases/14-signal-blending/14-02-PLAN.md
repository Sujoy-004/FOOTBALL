    **Part B — Build the calibration/blending pipeline in `_run_iteration()`:**

    Add a new private function after `_merge_signals_into_history()`:

    ```python
    def _calibrate_and_blend(teams, groups, bracket, odds_cache, cb_cache, elo_ratings) -> dict | None:
        """Flow A + Flow B: Fit calibration params, compute weights, blend per-match.
        
        Flow A: Fit calibration params from prediction_history for each signal.
        Flow B: Build match_probs dict by applying calibration + blending to each 
                match's per-signal probabilities.
        
        Returns blend_params dict with keys:
            "match_probs": dict[str, float] — match_id -> blended probability
        Or None if blending cannot be performed (no history, no signal data, etc.).
        
        Graceful degradation (D-22): never raises. Returns None when data insufficient.
        """
    ```

    Implementation details for `_calibrate_and_blend()`:

    1. **Load prediction_history** via `state.load_prediction_history()`
    2. **Fit calibration params per signal:**
       - For each signal key in ["elo", "market_odds", "catboost"]:
         - Collect (prediction, actual) pairs from history where signal is available
         - If len(pairs) >= COLD_START_THRESHOLD: fit via `calibrate_signal(preds, actuals)`
         - If below threshold: use (1.0, 0.0) for identity (D-03)
       - Store in `calibration_params` dict: `{signal_key: {"A": A, "B": B}}`
    3. **Persist calibration params** via `state.save_calibration_params(calibration_params)`
       (Also include n_matches and current brier for each signal in the persisted dict)
    4. **Compute per-signal rolling Brier** from history
       - Use `compute_metrics(preds, actuals)["brier"]` for each signal on the full history
       - Note: the rolling window is the full dataset when history < BRIER_WINDOW_SIZE
    5. **Compute blend weights** via `compute_blend_weights(signal_briers)` from D-07
    6. **Build match_probs** for all matches in signal caches:
       - Collect all match_ids from odds_cache and cb_cache
       - For each match_id, get per-signal probabilities:
         - elo: compute via `expected_score()` from teams' Elo ratings (need elo_ratings lookup)
         - market_odds: from odds_cache match entries
         - catboost: from cb_cache match entries
       - For each signal that has a probability for this match:
         - Apply calibration: `apply_calibration(p_raw, calibration_params[sig]["A"], calibration_params[sig]["B"])`
       - Blend calibrated probabilities using `blend_predictions(calibrated_probs, weights)`
       - Store: `match_probs[match_id] = blended_prob`
    7. **Return** `{"match_probs": match_probs}` or `None` if no data

    **Important:** The Elo probability for a match requires knowing both teams' Elo ratings. For group matches, the teams are known from groups.json. For knockout matches (R32+), the match_ids exist but the teams aren't known until simulation. The signal caches (odds_cache, catboost_cache) contain probabilities keyed by match_id for all matches, but the Elo probability can only be computed when teams are known.

    **Strategy for match_probs construction:**
    - For **group matches**: look up team_a, team_b from groups.json → compute elo probability → blend
    - For **R32+ knockout matches**: look up teams from bracket.json (if team_a/team_b are hardcoded) or skip if teams are dynamic (slot-based)
    - This means match_probs primarily covers group matches, which is correct per research recommendation (R32+ blending deferred)

    For the Elo probability computation of group matches:
    ```python
    # From groups.json data structure:
    for group_data in groups.get("groups", {}).values():
        for match in group_data["matches"]:
            mid = match["match_id"]
            ta, tb = match["team_a"], match["team_b"]
            elo_prob = expected_score(elo_ratings[ta], elo_ratings[tb])
    ```

    For bracket matches with hardcoded team_a/team_b:
    ```python
    for match in bracket:
        mid = match["match_id"]
        if match.get("team_a") and match.get("team_b"):
            elo_prob = expected_score(elo_ratings[match["team_a"]], elo_ratings[match["team_b"]])
    ```

    For team lookup from signal caches, the cache entries must contain team_a, team_b. If they don't, skip those entries for Elo probability computation.
