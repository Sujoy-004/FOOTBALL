    1. Function `load_teams(data_dir=None)`:
       - Accept optional `data_dir: Path | None`, default to `constants.DATA_DIR`
       - Open `data_dir / "teams.json"` with `open(..., encoding="utf-8")` and `json.load()`
       - Return `dict[str, dict]` (team_name → {"elo": int})
       - Let `FileNotFoundError` and `json.JSONDecodeError` propagate naturally (no catch-and-wrap)

    2. Function `load_bracket(data_dir=None)`:
       - Same pattern: open `data_dir / "bracket.json"`, `json.load()` → `list[dict]`
       - Call `validate_bracket(data)` BEFORE returning (validation happens on every load)
       - Return `list[dict]`

    3. Function `load_played(data_dir=None)`:
       - Same pattern: open `data_dir / "played.json"`, `json.load()` → `dict[str, dict]`
       - Return `dict[str, dict]`

    4. Function `validate_bracket(matches: list[dict]) -> None`:
       Follow RESEARCH.md Pattern 3 exactly:
       - **Step 1 — Uniqueness:** Iterate matches, build `match_ids: set[str]`. If any `match_id` repeats, `raise ValueError(f"Duplicate match_id: {mid}")`
       - **Step 2 — Reference integrity:** For each match, if `source_matches` is not None, check each source exists in `match_ids`. If missing, `raise ValueError(f"Match '{match_id}' references non-existent source_match '{src}'")`
       - **Step 3 — Cycle detection:** Build adjacency dict `adj: dict[str, list[str]]` where edges go from source → current match_id. Use 3-color DFS (`0=unvisited, 1=in-progress, 2=done`). If a node with color 1 is found during DFS, `raise ValueError(f"Circular dependency detected involving matches: {node}, {neighbor}")`
       
       Rules for validation:
       - `source_matches` can be `None` (for R16 matches) or a list of strings
       - Empty list `[]` is treated same as `None` — no references
       - Do NOT modify the input list
       - Raises `ValueError` only — no other exception type

    Use `from pathlib import Path`, `import json`, `from src import constants` (or `from src.constants import DATA_DIR`).
    All load functions use `pathlib.Path` for path construction.
